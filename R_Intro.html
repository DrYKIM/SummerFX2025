R Introduction: Basic Coding and Practice

Welcome to your introduction to R programming! R is a powerful language and environment for statistical computing and graphics. In this session, we'll cover how to access RStudio in Posit Cloud and then dive into some very basic R commands to get you started with data manipulation.

Part 1: Accessing RStudio in Posit Cloud (Free Version)

This section will guide you through setting up your RStudio environment.

Step 1: Sign Up / Log In to Posit Cloud

Open your web browser and go to posit.cloud. If you don't have an account, sign up for the free tier. You can usually sign up using a Google, GitHub, or Microsoft account for convenience. If you already have an account, simply log in.

Look for options like "Sign Up," "Get Started," or "Log In."

Step 2: Create a New Project

Once you're logged in, you'll see your Posit Cloud dashboard. Look for a button that says "New Project" (it might be a green button or in a prominent location). From the dropdown, select "New RStudio Project".

Posit Cloud will take a moment to spin up a new RStudio environment for you.

Step 3: Familiarize Yourself with RStudio

Once your project loads, you'll see the RStudio interface. It usually has four main panes:

  • Console (bottom-left): Where you type and run R commands.
  • Source Editor (top-left): Where you write and save your R scripts (.R files).
  • Environment/History (top-right): Shows loaded data, variables, and command history.
  • Files/Plots/Packages/Help (bottom-right): For managing files, viewing plots, installing packages, and accessing help.

Don't worry if it looks complex at first; we'll guide you through the essentials.

Part 2: Basic R Coding and Practice

Now that your RStudio environment is ready, let's write some basic R code! You can type these commands directly into the Console (bottom-left pane) and press Enter after each line, or type them into a new R Script (File > New File > R Script in the top-left pane) and run lines using Ctrl+Enter (Cmd+Enter on Mac).

Practice 1: Creating a Vector of Student Names

A vector is the most basic data structure in R. It's a sequence of data elements of the same basic type. Here, we'll create a vector for our group's student names. 


# Create a character vector for student names
student_names <- c("Christopher", "Gideon", "Christina", "Camille", "Sydney", "Boisseau")

# Print the vector to see its contents
print(student_names)
What to expect: The console will output the list of names you just created.

Practice 2: Creating a Factor for RStudio Experience

A factor is used to store categorical data in R. We'll create a factor to represent whether each student has used RStudio before.

Important: In the code block below, you will need to manually input the "Yes" or "No" answers for each student based on their responses during your session. Make sure the number of answers matches the number of students! 


# Manually input "Yes" or "No" answers for RStudio experience for each student
# Example: If Christopher said "Yes", Gideon "No", Christina "Yes", etc.
# rstudio_experience_raw <- c("Yes", "No", "Yes", "No", "Yes", "No") # Example input

# Replace the example above with your actual student responses:
rstudio_experience_raw <- c("Yes", "No", "Yes", "Yes", "No", "No") # <--- REPLACE WITH YOUR DATA

# Convert the raw answers into a factor with defined levels
# Defining levels ensures consistency and proper ordering if needed
rstudio_experience <- factor(rstudio_experience_raw, levels = c("Yes", "No"))

# Print the factor to see its contents and levels
print(rstudio_experience)
What to expect: You'll see the "Yes" or "No" answer for each student, followed by the "Levels" of the factor. Ensure the number of entries in `rstudio_experience_raw` matches the number of `student_names`.

Practice 3: Combining Data into a Data Frame

A data frame is a table-like structure where each column can contain a different type of data (e.g., names are text, experience is a category). It's the most common way to store data in R for analysis. 


# Combine the student names and RStudio experience into a data frame
# Each vector becomes a column in the data frame
student_data <- data.frame(
  Name = student_names,
  RStudio_Experience = rstudio_experience
)

# Print the data frame to see the combined table in the console
print(student_data)

# To view the data frame in a spreadsheet-like viewer (like Excel) in RStudio
View(student_data)
What to expect: The data frame will be printed in the console. Additionally, a new tab will open in RStudio's Source Editor pane, displaying `student_data` in a spreadsheet-like viewer, which is often more intuitive for seeing the data.

Practice 4: Creating a Frequency Table from the Data Frame

Finally, let's create a simple frequency table to see how many students have "Yes" or "No" RStudio experience. 


# Create a frequency table of RStudio experience
# table() counts the occurrences of each unique value in a factor
experience_table <- table(student_data$RStudio_Experience)

# Print the frequency table
print(experience_table)
What to expect: A table showing the counts for "Yes" and "No" RStudio experience.
Congratulations! You've just performed some fundamental data manipulation in R. This is the foundation for more complex analyses!
